# SimpleViewer

SimpleViewerはシンプルなWPFベースのイメージビューアです。
フォルダ、ZIP、PDF、単一の画像ファイルを開いてページ表示、見開き表示、サムネイルやツリーでのナビゲーションが可能です。

## 特徴

- フォルダ、ZIP、PDF、単一画像の読み込み
- ページ表示・見開き表示（単一表示・見開き右綴じ・見開き左綴じ）
- サムネイル表示とツリーナビゲーション（仮想化により多数ページでも高速）
- サムネイルとカタログ表示でのサムネイル共有（高速表示）
- ディスクキャッシュによるサムネイルの永続化
- ドラッグ＆ドロップでのファイル読み込み
- ズーム、パン（ドラッグ操作）、クリックによるページ遷移
- カタログオーバーレイ表示
- EXIF Orientation対応

## 主な操作

### マウス操作

- **画面左端（左20%）をクリック**：次のページへ進む
- **画面右端（右20%）をクリック**：前のページへ戻る
- **画像をドラッグ**：パン（スクロール）操作
- **ホイールスクロール**：ズームイン/ズームアウト（Ctrlキーを押しながら）
- **スライダー、サムネイル、ツリーパネル**：直接ページを指定して移動

### キーボードショートカット

#### ファイル操作
- `Ctrl+O` : ファイルを開く
- `Ctrl+W` : 現在のファイルを閉じる
- `Alt+F4` : アプリケーションを終了

#### 表示操作
- `S` : 表示モード切替（単一表示 ? 見開き右綴じ ? 見開き左綴じ）
- `F3` : カタログ表示切替
- `F4` : サイドバー表示切替

#### ページ移動
- `→` / `PageDown` : 次のページへ
- `←` / `PageUp` : 前のページへ
- `Home` : 最初のページへ
- `End` : 最後のページへ

#### ズーム操作
- `Ctrl++` : ズームイン
- `Ctrl+-` : ズームアウト
- `Ctrl+0` : 実寸表示（100%）
- `Ctrl+F` : ウィンドウ幅に合わせる
- `Ctrl+G` : ウィンドウ全体に合わせる
- `Ctrl+R` : ズームをリセット

> **注意**：クリック判定の端領域の幅（20%）や、ズーム倍率などの詳細設定はソースコード内で調整可能です。

## 必要条件

- Windows 10/11（WPFアプリケーションのため）
- .NET 8 SDK または .NET 8 Runtime

## ビルドと実行

### コマンドラインからビルド

1. リポジトリをクローンします:
   ```
   git clone https://github.com/NYSTM/SimpleViewer.git
   cd SimpleViewer
   ```

2. 依存関係を復元してビルドします:
   ```
   dotnet restore
   dotnet build -c Release
   ```

3. 実行します:
   ```
   dotnet run --project SimpleViewer\SimpleViewer.csproj
   ```

### Visual Studio でビルド

1. Visual Studio 2022 以降でソリューションファイル（`.sln`）を開きます。
2. ビルド構成を `Release` に設定します。
3. `F5` キーを押してデバッグ実行、または `Ctrl+F5` キーでデバッグなしで実行します。

## 使い方

1. **SimpleViewerを起動**します。

2. **ファイルを開く**:
   - メニューバーから「ファイル」→「開く」を選択
   - または `Ctrl+O` キーを押す
   - またはファイル/フォルダを**ドラッグ＆ドロップ**

3. **対応ファイル形式**:
   - **フォルダ**: フォルダ内の画像ファイルを順次表示
   - **ZIP**: ZIP内の画像ファイルを展開せずに表示
   - **PDF**: PDFファイルの各ページを画像として表示
   - **画像ファイル**: JPEG, PNG, BMP, GIF, TIFF など

4. **表示モードの切り替え**:
   - メニューバーから「表示」→「表示モード切替」
   - または `S` キーを押す
   - 単一表示 → 見開き右綴じ → 見開き左綴じ → ...（循環）

5. **ナビゲーション**:
   - **サムネイル**: 左側のサイドバーに自動生成されるサムネイルをクリック
   - **ツリー**: ファイル構造をツリー表示してナビゲート
   - **カタログ**: `F3` キーでカタログビューを表示し、一覧から選択

6. **終了**:
   - メニューバーから「ファイル」→「終了」
   - または `Alt+F4` キーを押す

## サムネイルキャッシュについて

SimpleViewerは、サムネイルを以下のようにキャッシュして高速表示を実現しています：

- **メモリキャッシュ**: 直近のサムネイルをメモリに保持
- **ディスクキャッシュ**: `cache` フォルダ内にソースごとのサブフォルダを作成し、サムネイルを永続化
- **自動削除**: ソースを切り替えると、以前のソースのキャッシュは自動的に削除（セキュリティ重視）

キャッシュの設定は、設定ファイル（`settings.json`）で変更可能です。

## 設定ファイル

SimpleViewerの動作は、実行ファイルと同じフォルダ内の `settings.json` で設定できます。

主な設定項目:
- **サムネイルキャッシュサイズ**: ディスクキャッシュの最大サイズ（MB）
- **セキュア削除**: キャッシュ削除時にファイルを上書きしてから削除
- **EXIF Orientation適用**: 画像の向き情報を自動適用

## トラブルシューティング

### サムネイル表示が遅い
- ディスクキャッシュが有効になっているか確認してください
- 初回表示時はサムネイル生成に時間がかかりますが、2回目以降はキャッシュから高速表示されます

### 画像が表示されない
- 対応ファイル形式か確認してください
- PDFの場合、PDFiumライブラリが正しくロードされているか確認してください

### 表示が乱れる
- ズームをリセット（`Ctrl+R`）してみてください
- ウィンドウサイズを変更してみてください

## 開発者向け情報

### アーキテクチャ

- **MVP パターン**: プレゼンテーションロジックとビューを分離
- **非同期処理**: すべての I/O 操作は非同期で実行
- **バッチ処理**: サムネイル生成は並列バッチ処理で高速化
- **キャッシュ階層**: メモリキャッシュとディスクキャッシュの2層構造

### 主要コンポーネント

- `SimpleViewerPresenter`: メインのプレゼンテーションロジック
- `ThumbnailService`: サムネイル生成とキャッシュ管理
- `DiskCacheManager`: ディスクキャッシュの管理（ソースごとのサブフォルダ）
- `ThumbnailController`: サムネイルバーの表示制御
- `CatalogController`: カタログ表示の制御
